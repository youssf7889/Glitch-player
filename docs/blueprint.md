# **App Name**: Glitch Player

## Core Features:

- Track Upload: Allow users to upload individual audio files (mp3, wav, etc.) for playback.
- Playlist Creation & Management: Enable users to create new playlists, rename existing ones, and delete playlists.
- Playlist Content Management: Support uploading entire folders as playlists and adding, removing, and reordering tracks within playlists.
- Player Controls: Implement standard music player functionalities: play, pause, next track, previous track, seek bar, volume control, repeat, and shuffle.
- Drag-and-Drop Interaction: Facilitate reordering tracks within playlists and managing playlist content using intuitive drag-and-drop actions.
- Local Session Persistence: Persist user playlists and the last active player state locally using IndexedDB/LocalStorage for a seamless experience.
- Keyboard Shortcuts: Implement optional keyboard shortcuts for essential player actions like play/pause (spacebar) and skip (arrow keys).

## Style Guidelines:

- Primary color: A warm, deep orange-brown (#AA7442) for interactive elements, highlights, and main borders, echoing the retro aesthetic.
- Background color: A pale, desaturated orange-beige (#FBF4EE) for the main application canvas, maintaining a cohesive, warm and light feel.
- Accent color: A rich, deep crimson-brown (#603C3C) to provide strong contrast for outlines, select text, and subtle differentiation of UI components, while remaining analogous to the primary palette.
- All text elements will use a monospace pixel font to precisely replicate the 8-bit digital aesthetic shown in the design. Note: currently only Google Fonts are supported.
- Custom-designed pixel art icons will be used throughout the application to strictly match the retro, digital visual style depicted in the image.
- The application's layout will exactly replicate the spatial arrangement, proportions, and element sizing from the provided image, including precise replication of list highlighting and container structures, ensuring responsiveness.
- Minimal interaction animations will be employed, primarily subtle state changes or feedback animations, to align with the retro, low-fidelity feel of the original design.
